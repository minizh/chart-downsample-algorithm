# ECharts 海量数据性能优化方案汇总

## 1. 性能瓶颈本质

### 渲染引擎差异
| 维度 | SVG 模式 | Canvas 模式 |
|------|----------|-------------|
| **原理** | DOM 元素渲染，每个数据点对应独立 DOM 节点 | 像素绘制，单一 Canvas 元素 |
| **1万点** | 1200ms | 350ms |
| **5万点** | 卡顿严重 | 900ms |
| **10万点** | 页面崩溃 | 1800ms |

**关键决策**：大数据场景（>1000点）必须显式指定 `renderer: 'canvas'`。

### 内存占用三大来源
- **原始数据存储**：JS 对象数组，含多维信息（坐标、颜色、大小等）
- **派生数据计算**：屏幕坐标转换、样式计算中间结果
- **可视化元素内存**：SVG 每个元素约 500 字节

### 计算复杂度陷阱
坐标转换、视觉编码、数据聚合均为 O(n) 或 O(n²)，地理等值线插值在 n>10,000 时可达数秒。

## 2. 核心优化策略体系：从基础到高级

### 2.1 数据层：从源头减量
- **智能采样**：LTTB（最大三角形三桶）算法保留趋势特征，或等距采样简单高效
- **数据分片与懒加载**：按可视区域异步加载，配合 `requestIdleCallback`
- **TypedArray 存储**：用 `Float32Array`/`Uint8Array` 替代普通数组，减少内存碎片

### 2.2 渲染层：极致调优
**ECharts 内置优化配置**：
```javascript
{
  large: true,              // 启用大数据模式
  largeThreshold: 2000,     // 阈值
  progressive: 500,         // 每帧渲染数据量
  progressiveThreshold: 3000,
  animation: data.length < 2000,  // 大数据关动画
  sampling: 'max',          // 升采样
  symbol: 'none'            // 关闭数据点符号
}
```

**增量渲染**：使用 `appendData` 方法配合 `requestAnimationFrame`，避免阻塞主线程。

### 2.3 内存层：防止泄漏
- **正确销毁**：调用 `chart.dispose()` 释放内部资源，避免重复创建实例
- **对象复用**：复用图表实例而非重新 `init`
- **内存监控**：通过 `performance.memory` 检测，超阈值触发清理

## 3. 进阶方案

### Web Worker 离屏处理
将 LTTB 采样、数据过滤、聚合计算移至 Worker 线程，通过可转移对象（Transferable Objects）快速且高效的零拷贝操作在上下文之间移动内存里的资源。

```javascript
1. 关闭数据点符号 (showSymbol: false)
2. 减少/关闭动画
3. 提高采样率（减少数据点）
4. 切换为更简单的图表类型
```

### 脏矩形优化 (`useDirtyRect: true`)
仅重绘变化区域，减少无效渲染。

## 4. 关键优化清单

| 数据规模 | 必需优化措施 |
|---------|------------|
| > 1000 | Canvas 渲染器 |
| > 5000 | `large` 模式 + 渐进式渲染 |
| > 50000 | 数据分片 + 增量加载 + Web Worker 预处理 |
| 实时流 | `appendData` 增量更新  + Web Worker + 数据过期机制 + 智能聚合 |
| 内存敏感场景 |  采用 TypedArray 和对象复用 |

---

## 5. 总结：核心要点

**Q: 百万级数据如何优化？**
> 分层优化：

1.  数据层用 LTTB 采样将 原始数据量降至 5 万条以内，Web Worker 离屏处理

2.  渲染层启用 `large` 模式 + 渐进式渲染，Canvas 渲染器将耗时控制在 3s 以内

3.  内存层用 TypedArray 减少 40% 占用，最终实现 60FPS。

**Q: 性能指标量化：如何精确定位或监控性能问题？**
> Chrome Performance 面板分析 Layout/Paint 阶段， 利用 ECharts `rendered`事件和 Performance API 精确测量渲染耗时，自建 FPS 与内存监控面板，自动触发降级策略。

1.  当检测到性能下降时自动触发降级策略，比如关闭动画、简化图形元素等
2.  快速定位到是数据预处理、渲染计算还是内存回收导致的性能问题

**Q: 实时更新场景**
> 增量更新 (`appendData`) + 智能聚合（根据缩放级别动态调整粒度）+ 数据过期机制 + 根据设备或较低版本浏览器情况自动降采样。

---

## 6. 调试面板关键指标

- **数据点数**：实时显示当前渲染点量
- **渲染时间**：单次 `setOption` 耗时（目标 < 16ms）
- **FPS**：实时帧率监控（低于 30 触发告警）
- **内存占用**：JS Heap 大小（超过 500MB 触发清理）
- **渲染器类型**：确认 Canvas/SVG 状态

---

**核心原则**：优化是持续监测→分析→调整的闭环，非一次性工作。数据预处理减轻计算压力，Canvas/渐进式渲染减轻 GPU 压力，内存管理防止泄漏，三者结合实现 Echarts 海量数据的流畅渲染。
